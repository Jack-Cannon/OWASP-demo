This page simulates a user profile viewer. In a real application, you'd typically only be able to view your own profile.

The vulnerability: There's no backend check to ensure the user requesting the profile ID is authorized to view it.

ðŸ’¡ How to Demonstrate the Vulnerability:

  1. Enter a user ID (e.g., "123") and click "View Profile".
  2. Now, change the User ID to a different number (e.g., "124" or "125") and click "View Profile" again.
  3. **Observation:** You can view any user's profile data by simply changing the ID, without any authentication or authorization checks.
  4. **Real-world impact:** An attacker could iterate through IDs to scrape sensitive data from many users.

The Fix: Server-side logic should always verify that the authenticated user has permission to access the requested resource (e.g., `if (loggedInUser.id === requestedUserId || loggedInUser.isAdmin) { ... }`).
